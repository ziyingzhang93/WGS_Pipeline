1. What is sarek?

In one sentence: nf-core/sarek is a complete WGS/WES variant calling pipeline — written in Nextflow — that takes raw sequencing data (FASTQ) and produces annotated variant calls (VCF) and quality reports in a single run.

一句话介绍：nf-core/sarek 是一个完整的全基因组/全外显子组测序变异检测流程，用 Nextflow 编写，可以从原始测序数据（FASTQ）一步到位产出注释好的变异结果（VCF）和质量报告。
  • End-to-end automation — FASTQ to annotated VCF in one command
    端到端自动化 — 一条命令从FASTQ到注释VCF
  • Multiple variant callers — HaplotypeCaller, DeepVariant, Mutect2, Manta, Strelka2, etc.
    支持多种变异检测工具 — HaplotypeCaller、DeepVariant、Mutect2、Manta、Strelka2 等
  • Efficient parallelization — splits genome into chunks for fast analysis
    高效并行处理 — 将基因组分块处理，大幅加速分析
  • Production-grade quality — maintained by the nf-core community, rigorously tested
    生产级质量 — nf-core 社区维护，经过严格测试
  • Highly portable — runs locally, on HPC clusters, or in the cloud (AWS/GCP)
    可移植性强 — 支持本地、集群、云平台（AWS/GCP）运行

2. WGS Pipeline Overview

Complete Pipeline Flowchart

完整流程图
1
Quality Control — FastQC
Assess sequencing quality and detect biases before processing.
Input: FASTQ  →  Output: HTML QC report
质量控制 — 在处理前评估测序质量，检测偏差。评估测序是否成功、数据质量如何。
▼
2
Adapter Trimming — FASTP
Remove sequencing adapters and low-quality reads to clean the data.
Input: FASTQ  →  Output: Cleaned FASTQ
去接头 — 去除测序接头序列和低质量reads，清洁数据。接头是测序过程中人工引入的短序列。
▼
3
Read Alignment — BWA-MEM / BWA-MEM2
Map each sequencing read to its location on the reference genome. Like matching puzzle pieces — find where each DNA fragment belongs.
Input: FASTQ + Reference  →  Output: BAM file
序列比对 — 将每条测序read比对到参考基因组的对应位置。就像拼拼图——找到每条DNA片段在基因组中的位置。
▼
4
Mark Duplicates — MarkDuplicates
Identify and flag PCR duplicate reads. During library prep, some DNA fragments get amplified more than others — these duplicates can bias variant calling.
Input: BAM  →  Output: Deduplicated BAM
标记重复 — 识别并标记PCR重复reads。在建库过程中，某些DNA片段会被过度扩增——这些重复会导致变异检测产生偏差。
▼
5
Base Quality Score Recalibration — BQSR
Correct systematic errors in the quality scores assigned by the sequencing machine using known variant databases (dbSNP, Mills, 1000G).
Input: BAM  →  Output: Recalibrated BAM
碱基质量校正 — 使用已知变异数据库（dbSNP、Mills、1000G）校正测序仪产生的系统性质量评分偏差。
▼
6
Variant Calling — Multiple tools available
Detect genetic variants (SNPs, InDels, structural variants) from the aligned reads.
  • HaplotypeCaller — local reassembly + Bayesian inference (recommended for SNP/InDel)
  • DeepVariant — deep learning CNN from pileup images (highest accuracy)
  • Manta — structural variant detection (large rearrangements)
  • Mutect2 — somatic variant calling (tumor mutations)
Output: gVCF and VCF files
变异检测 — 从比对结果中检测遗传变异（SNP、InDel、结构变异）。
· HaplotypeCaller — 局部重组装 + 贝叶斯推断（推荐用于SNP/InDel）
· DeepVariant — 用CNN深度学习从reads堆叠图像推断（准确度最高）
· Manta — 结构变异检测（大型基因组重排）
· Mutect2 — 体细胞变异检测（肿瘤突变）
▼
7
Joint Genotyping — GenotypeGVCFs (optional)
Re-genotype all samples together to improve accuracy. Only needed when analyzing multiple samples as a cohort.
Input: gVCF  →  Output: Merged VCF
联合分型 — 将所有样本放在一起重新分型，提高准确度。仅在多样本队列分析时需要。
▼
8
Variant Annotation — VEP / SnpEff
Annotate each variant with biological significance: coding/non-coding region, known databases (gnomAD, ClinVar), predicted impact (missense, frameshift, etc.).
Output: Annotated VCF
变异注释 — 标注每个变异的生物学意义：编码区/非编码区、已知数据库（gnomAD、ClinVar）、预测影响（错义突变、移码突变等）。
▼
9
Summary Report — MultiQC
Aggregate all QC metrics into a single interactive HTML report: alignment rate, coverage depth, variant counts and distribution.
Output: Interactive HTML report
质控总报告 — 汇总所有QC指标为一份交互式HTML报告：比对率、覆盖深度、变异数量和分布。

5. Core Concepts Explained

核心概念解释 — 用通俗语言解释Nextflow的关键术语

Channel (Data Stream)

数据流 — 想象一条流水线上的传送带，样本就是传送带上的工件

A channel is how data flows between steps. Think of it as a conveyor belt in a factory — samples move along automatically, and each workstation (process) picks up items, processes them, and sends them forward.

Channel 是步骤之间传递数据的方式。想象工厂的传送带——样本自动移动，每个工位（process）取出物品、处理后送往下一站。关键点：Channel是异步的——不需要等一个样本完成才开始下一个，1000个样本可以同时处理。
workflow FASTQ_ALIGN {
    take:
    reads       // Input: FASTQ files arrive on this channel
    index       // Input: reference genome index

    main:
    BWAMEM1_MEM(reads, index, sort)   // Process the data

    emit:
    bam = BWAMEM1_MEM.out.bam        // Output: BAM files leave on this channel
}

Process (Execution Unit)

执行单元 — 一个独立的任务工作室，每次处理一个样本

A process is the smallest execution unit — it wraps a single tool's command line. Each process runs in an isolated environment (Docker container or Conda) with defined CPU/memory limits.

Process 是最小的执行单元——封装一个工具的命令行。每个process在隔离环境中运行（Docker容器或Conda），有明确的CPU/内存限制。
process GATK4_HAPLOTYPECALLER {
    tag "$meta.id"            // Label: sample ID for log tracking
    label 'process_low'       // Resources: low CPU & memory
    container 'gatk4:...'     // Environment: Docker image with GATK4

    input:
    tuple val(meta), path(bam)        // Input: sample metadata + BAM file

    output:
    tuple val(meta), path("*.vcf.gz") // Output: sample metadata + VCF file

    script:
    """
    gatk HaplotypeCaller --input $bam --output ${meta.id}.vcf.gz ...
    """
}

Subworkflow (Reusable Step Collection)

子流程 — 一个可复用的"小工厂"，把多个process组合成一套完整的工作步骤

A subworkflow groups multiple processes into a reusable pipeline segment. For example, the fastq_align subworkflow bundles the alignment tool call, output handling, and version tracking into one clean unit.

Subworkflow 将多个process组合成可复用的流程片段。例如 fastq_align 子流程把比对工具调用、输出处理和版本追踪打包成一个整洁的单元。

Meta Map (Sample Identity Card)

样本元信息 — 每个样本的"身份证"，记录所有属性

Every piece of data carries a meta map — a dictionary of sample attributes that travels with the file through the entire pipeline. This ensures you always know which sample a file belongs to.

每条数据都携带一个meta map——一个包含样本属性的字典，伴随文件贯穿整个流程。确保你始终知道每个文件属于哪个样本。
[
    id: 'patient1_tumor',       // Sample ID
    patient: 'patient1',        // Patient ID
    status: 1,                  // 0 = normal, 1 = tumor
    sex: 'XX',                  // Sex
    data_type: 'cram',          // File format
    variantcaller: 'haplotypecaller'  // Tool used
]

Key Biology Concepts

几个关键的生物学概念

SNP vs InDel

SNP（单核苷酸多态性）vs InDel（插入/缺失）
SNP: Single base change (most common variant type)
ref:  A T G C C G
alt:  A T A C C G    ← C→A substitution

InDel: Insertion or deletion of bases
ref:  A T G C C G
alt:  A T G - - G    ← deletion of CC
alt:  A T G C T C C G  ← insertion of TC

Genotype Notation

基因型表示法
GenotypeMeaning
0/0Homozygous reference — both chromosomes carry the reference base ("wild type")
纯合参考 — 两条染色体都是参考碱基（"野生型"）
0/1Heterozygous — one reference, one variant allele (carrier)
杂合 — 一条参考，一条变异（携带者）
1/1Homozygous alternate — both chromosomes carry the variant
纯合变异 — 两条染色体都是变异碱基

gVCF vs VCF

VCF contains only variant sites. gVCF (Genomic VCF) includes all sites — even non-variant ones. Why? For multi-sample joint genotyping, you need to know "no variant was found here" to accurately assess consistency across samples.

VCF 只包含变异位点。gVCF（Genomic VCF）包含所有位点——包括没有变异的。为什么？做多样本联合分型时，需要知道"这里没有发现变异"，才能准确评估样本间的一致性。

Coverage Depth

覆盖深度
DepthMeaning
<5×Unreliable — too few reads to confidently call variants
不可信 — reads太少，无法准确检测变异
20–100×Good coverage — standard for WGS (30× is typical target)
良好覆盖 — WGS标准（30×是常用目标深度）
>200×Suspiciously high — may indicate repetitive region or contamination
异常高 — 可能是重复区域或污染

8. Three Variant Calling Modes

三种变异检测模式 — sarek根据样本类型自动选择分析路径

A. Germline Mode — Most Common

生殖系模式 — 最常用

Use case: Detecting inherited variants in a single individual. Used in population genetics, family studies, hereditary disease research.

适用场景：检测个体遗传变异。用于群体遗传学、家系研究、遗传性疾病研究。

What it detects: Variants present in all cells (inherited from parents). Expected heterozygosity: ~50%.

检测内容：存在于所有细胞中的变异（从父母遗传）。预期杂合度约50%。

Tools: --tools haplotypecaller or --tools deepvariant

In the code: The main workflow checks meta.status == 0 (normal sample) and routes to BAM_VARIANT_CALLING_GERMLINE_ALL.
在代码中：主流程检查 meta.status == 0（正常样本），然后路由到 BAM_VARIANT_CALLING_GERMLINE_ALL。

B. Somatic Paired Mode — Cancer Research

体细胞配对模式 — 癌症研究

Use case: Tumor + matched normal tissue from the same patient. Identifies mutations unique to the tumor (somatic mutations / driver mutations).

适用场景：同一患者的肿瘤 + 配对正常组织。识别肿瘤特有的突变（体细胞突变/驱动突变）。

How it works: Compares tumor BAM vs normal BAM. Variants found only in the tumor are called somatic.

工作原理：比较肿瘤BAM和正常BAM。只在肿瘤中发现的变异被认定为体细胞突变。

Tools: --tools mutect2, strelka2, manta

In the code: The workflow uses .cross() to pair tumor (status == 1) with normal (status == 0) by patient ID, then routes to BAM_VARIANT_CALLING_SOMATIC_ALL.
在代码中：流程使用 .cross() 按患者ID将肿瘤样本（status==1）和正常样本（status==0）配对，然后路由到 BAM_VARIANT_CALLING_SOMATIC_ALL。

C. Tumor-Only Mode — Clinical Diagnostics

仅肿瘤模式 — 临床诊断

Use case: Only tumor sample available, no matched normal. Common in clinical settings where cost is a concern.

适用场景：只有肿瘤样本，没有配对正常对照。常见于成本受限的临床场景。
Limitation: Cannot distinguish inherited variants from somatic mutations without a normal control. Requires stricter filtering (population frequency databases like gnomAD are essential).
限制：没有正常对照无法区分遗传变异和体细胞突变。需要更严格的过滤（群体频率数据库如gnomAD不可或缺）。

Tools: --tools mutect2

In the code: Tumor samples with no matched normal are identified via .filter { !(it.last()) } and routed to BAM_VARIANT_CALLING_TUMOR_ONLY_ALL.
在代码中：没有配对正常的肿瘤样本通过 .filter { !(it.last()) } 识别，路由到 BAM_VARIANT_CALLING_TUMOR_ONLY_ALL。

9. Example Commands

工作流执行命令示例

Basic Germline Analysis

基础生殖系分析
nextflow run nf-core/sarek \
  --input samplesheet.csv \
  --genome GATK.GRCh38 \
  --tools haplotypecaller \
  --outdir results/ \
  -profile docker

Germline with DeepVariant (higher accuracy)

使用DeepVariant的生殖系分析（更高准确度）
nextflow run nf-core/sarek \
  --input samplesheet.csv \
  --genome GATK.GRCh38 \
  --tools deepvariant \
  --outdir results/ \
  -profile docker

Somatic Paired Analysis (cancer)

配对体细胞分析（癌症）

Samplesheet format — use status column to indicate normal (0) vs tumor (1). Samples are paired automatically by patient ID.

Samplesheet 格式 — 用 status 列标注正常组织（0）和肿瘤组织（1）。流程会自动按 patient ID 配对。
patient,sample,lane,fastq_1,fastq_2,status
patient1,normal_sample,lane1,normal_R1.fastq.gz,normal_R2.fastq.gz,0
patient1,tumor_sample,lane1,tumor_R1.fastq.gz,tumor_R2.fastq.gz,1
nextflow run nf-core/sarek \
  --input samplesheet.csv \
  --genome GATK.GRCh38 \
  --tools mutect2,manta,strelka2 \
  --outdir results/ \
  -profile docker

Resume after interruption

中断后断点续跑
nextflow run nf-core/sarek \
  --input samplesheet.csv \
  --tools haplotypecaller \
  -resume

Quick test with small dataset

用小数据集快速测试
nextflow run nf-core/sarek -profile docker,test
